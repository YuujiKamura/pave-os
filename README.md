# pave-os

完成した土木工事プロジェクトのフォルダを食わせると、構造化されたナレッジベース（JSON）を生成する。

次の工事で「前回どうだった？」に答えるための参照データを自動で作る。

## 背景

土木工事（特に舗装）では、1案件あたり2000〜3000ファイルが20以上のフォルダに散在する。入札から竣工まで半年〜1年。フォルダ番号は時系列ではなく「必要になった順」に作られる。

この構造から「何がいつ、どういう順で、何と並列に走ったか」を人間が読み取るのは難しい。しかしAIにファイルメタデータを渡せば、フェーズ分類・並列タスク特定・成果物集計を一発で生成できる。

### 着想元

[ShipOS](https://www.palantir.com/shipos/)（Palantir × 米海軍）— バラバラな造船所のデータをAI層で統合し、工程計画を160時間の手作業から10分未満に短縮した。pave-osは同じ思想を土木工事に適用する。

## 使い方

```bash
./analyze_project.sh "H:/マイドライブ/〇〇舗装補修工事"
```

出力: `output/YYYYMMDD_HHMMSS_工事名.json`

### 必要なもの

- bash（Git Bash可）
- [Gemini CLI](https://github.com/google-gemini/gemini-cli)（認証済み）
- `find` コマンド

### 環境変数

| 変数 | デフォルト | 説明 |
|------|-----------|------|
| `PAVEOS_MODEL` | `gemini-2.5-flash` | 使用するGeminiモデル |

## 仕組み

1. `find` で全ファイルのメタデータ（日時・サイズ・パス）を1回で取得
2. `awk` でフォルダごとに集計・圧縮（2700ファイル → 9KB）
3. Gemini CLIに圧縮済みコンテキストを投げてJSON生成

Claude等の有料APIを消費しない。Gemini CLIで完結する。
